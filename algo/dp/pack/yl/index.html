    有依赖的背包¶

    问题描述¶

    给出的物品间存在某种 依赖关系，类似 拓扑序

    比如你去旅游要带手机，就必须带充电器，否则你就不能带手机

    规定：每件物品最多只能依赖一件物品

    方法¶

    树形 DP¶

    规定：每件物品最多只能依赖一件物品

    我们很自然地会联想到树的性质：

    1. 每个结点最多只有一个父结点

    2. 两个物品间的依赖关系则可以描述为 选子结点必须先选父结点

    3. 一件物品能够被选的充要条件为 它到根结点的路径上的所有结点都已经被选

    不过还有一个问题，如果 不依赖任何其他物品的物品 不止一个，那得到的就是一个森林而不是一棵树，我们虚拟一个 0 号结点 ，并将其作为 不依赖于任何其他物品的物品 的父结点，这样就得到了一棵以 0 号结点为根的树

    记 f[u][v] 表示编号为 u 的当前结点 取 j 个子结点 所能达到的最大总价值，v 表示当前儿子

    决策：取 / 不取当前儿子 v

    1. 不取当前儿子，则继续遍历下一个儿子，f[u][j] 保持不变

    2. 取当前儿子，则 j 要加上 所取儿子所取的子结点数 k ，价值 f 要加上 所取儿子的价值 f[v][k]

    状态转移方程如下：

    f[u][j]=max(f[u][j],f[u][j-k]+f[v][k])

    考虑枚举顺序，更新 f[u][j] 时 f[u][j-k] 和 f[v][k] 中应该都是上一个儿子的状态，所以应该逆序枚举 j

    Note

    为了方便理解，上面给出的状态转移方程已经用滚动数组优化掉了一维（遍历儿子那一维）
