    封装的权限控制¶

    公共权限¶

    关键字：public

    成员 类内可以访问 类外也可访问

    保护权限¶

    关键字：protected

    成员 类内可以访问 类外 不可以 访问

    父亲的保护内容 儿子可以访问

    私有权限¶

    关键字：private

    成员 类内可以访问 类外 不可以 访问

    父亲的私有内容 儿子 不可以 访问

    Warning

    struct 默认访问权限为公共，而 private 则为私有，除此之外两者完全相同

    构造函数 & 析构函数¶

    定义语法¶

    构造函数：类名(){}

    析构函数：~类名(){}

    Example

    1
2
3
4
5
6
7
8
9
    struct info{
public:
    info(){
        // 构造函数
    }
    ~info(){
        // 析构函数
    }
};

    Warning

    构造函数 & 析构函数都要放在 public 下

    浅拷贝¶

    编译器默认提供，是简单的值拷贝

    如果浅拷贝作用于堆区对象，被拷贝的就只有指向对象的指针，而不会拷贝内存，也就是说原对象与拷贝对象共用一块内存，这样析构时会这块内存会被释放两次（重复释放）导致错误
