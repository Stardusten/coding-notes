    静态成员¶

    静态成员变量¶

    1. 存于全局区（不占用类空间，编译阶段分配内存，所有对象共用一份数据）
    2. 类内声明，类外初始化
    3. 既可通过对象访问，也可通过类名访问

    class person{
public:
    static int m_A; // 声明静态成员变量 m_A
}
int person::m_A=100; // 类外初始化
void test(){
    person p1;
    cout<<p1.m_A<<endl; // 通过对象访问
    // Result: 100
    person p2;
    p2.m_A=200;
    cout<<p1.m_A<<' '<<p2.m_A<<endl;
    // Result: 200 200
    // 注意 p1，p2 共用一份数据，牵一发而动全身
    cout<<person::m_A<<endl; // 通过类名访问 
}

    3. 既可通过对象访问，也可通过类名访问

    class person{
public:
    static void func(){ // 声明静态成员函数
        cout<<"静态成员函数 func 调用"<<endl;
        m_A+=10; // 静态成员函数可以调用静态成员变量
        m_b+=10;  // 错误，静态成员函数不能调用普通成员变量
    }
    static int m_A;
    int m_B;
}
int person::m_A=0;
void test(){
    person p1;
    cout<<p1.m_A<<endl;
    // Result: 0
    p1.func(); // 通过对象调用
    // Result: 静态成员函数 func 调用
    cout<<p1.m_A<<endl; 
    // Result: 10
    person::func(); // 通过类名调用
    // Result: 静态成员函数 func 调用
    cout<<person::m_A<<endl; 
    // Result: 20
}

    this 指针¶

    实质¶

    隐藏在成员函数内部的 指针常量 ClassName * const this，成员函数内的所有成员变量都由 this 指针调用

    Note

    指针常量：指向 不可修改（指向 A 后就不能改为指向 B 了）

    常量指针：所指对象 不可修改

    作用¶

    1. 防止命名冲突（如：形参名与成员名冲突）
    2. 非静态成员函数中返回调用对象本身（使用 return *this ，可实现链式调用）

    class person{
public:
    void clear(){
        age=0; // 即 this->age=0;
    }
    person& add_age(int age){ // 注意使用引用
        this->age+=age; // 解决形参与成员命名冲突
        return *this; // 返回调用对象本身
    }
    int age;
}
void test(){
    person p1(10);
    cout<<p1.age<<endl;
    // Result: 10
    p1.add_age(10).add_age(20).add_age(30); // 链式调用
    cout<<p1.age<<endl;
    // Result: 70 (10+10+20+30)
}

    class person{
public:
    void edit(){ const // 修饰 edit() 函数为常函数
        m_A=100; // 即 this->m_A=100;
        // 错误，常函数内不能修改成员变量值
        m_B=100；
        // mutable 修饰后，m_B 在常函数内也可修改
    }
    int m_A;
    mutable int m_B; // 特殊变量，即使在常函数中也可修改
};
